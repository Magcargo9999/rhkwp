### 문제1 stakeholder란 무엇이고 stakeholder가 증가할 경우에 발생하는 문제점에 대해서 "추상화와 모델링" 이 어떠한 효과가 있는지 설명하시오.
 * stakeholder는 하나의 프로젝트에 이해관계를 가진 이해관계자를 말합니다.
 * 한 프로젝트에는 목표를 제시하는 고객, 제품을 사용하는 사용자, 요구사항을 정리하는 기획자, 기능을 구현하는 개발자, 그리고 배포와 유지보수를 담당하는 운영자 등 수많은 이해관계자(Stakeholder)가 존재합니다. 이해관계자가 늘어날수록 소통 채널의 수는 기하급수적으로 증가하여 원활한 소통이 어려워집니다. 특히 이들은 서로 다른 관점으로 프로젝트를 바라보기 때문에, 소통이 어긋나면 결국 고객의 의도와는 다른 결과물이 나올 수 있습니다.

![stakeholder_complexity](https://github.com/user-attachments/assets/85925a22-aff3-4284-a4e9-73db1e147879)

 
 * 추상화와 모델링의 효과
   * 추상화를 통하여 본질적이지 않은 부분을 제거하고 핵심적인 부분만 추출하여 단순화시키는 것으로, stakeholder들이 프로젝트에 관해 이해를 쉽게 하도록 돕습니다.
   * 추상화한 부분을 모델링하여 설계 도면을 만들고, 이를 통해 이해관계자들이 같은 관점으로 이해하도록 합니다.

 ### 문제2 소프트웨어의 개발기간과 이용기간 장기화에 의해 발생하는 문제점에 대해서 "추적가능성"이 어떠한 효과기 있는지 설명하시오.
* 개발 기간이 장기화된다면 해당 소프트웨어를 어떠한 기준으로 왜 이렇게 프로그래밍 한 것인지 알기 힘들어집니다.
  초기 개발자가 퇴사하게 된다면 코드의 핵심 개념과 코드가 어떻게 만들어져 왔는지를 아는 사람이 없어질 수도 있습니다.
  
* 이용기간이 장기화될시 발생하는 문제점은 잦은수정으로 인해 코드가 지저분해져
  처음 설계 단계에서의 프로그램과 오랜 이용 기간이 지난 후의 프로그램의 형태가 많이 달라질 수도 있습니다.

* 추적가능성은 프로그램을 개발하는도중에 일어난일들을 문서로 남겨서 추적이 가능하게하는 것인데 이런식으로 설계도면을 남겨두면 소프트웨어의 개발기간과 이용기간이
  장기화됨에따라서 왜 그런방식으로 소프트웨어를 개발했는지를 몰라서 생기는 문제들을 해결할수있고 나중에 디버깅을 하거나 코드를 수정할때도 훨씬 들이는 리소를 줄   일수 있게됩니다.
